# Dashboard
Projeto nosso Cyberkayzen de Dashboard

A ENTREGA DEVE CONTER:
1. Dashboard funcional
O grupo deverá construir um dashboard digital funcional em uma plataforma de sua escolha. Exemplos de plataformas possíveis: Power BI; Looker Studio; Tableau; Streamlit; sites em HTML/CSS/JavaScript; aplicações web; outras ferramentas equivalentes.

2. Estrutura mínima obrigatória
O dashboard deverá apresentar: indicadores de atraso; visualização das entregas atrasadas; comparação entre transportadoras; análise por região; algum tipo de priorização visual dos maiores problemas operacionais.

3. Recursos de análise
A solução deverá possuir recursos que facilitem interpretação e análise dos dados, como:
filtros; gráficos; tabelas dinâmicas; rankings; alertas visuais; destaques automáticos; organização estratégica das informações.

O squad deve considerar que a base apresentada é uma amostra simplificada para fins didáticos. Será avaliada a organização, visualização e interpretação dos dados em um dashboard, e não a realização de análises estatísticas sobre grandes volumes de informação.

4. Forma de entrega e avaliação (REQUISITO CRÍTICO)
O grupo deverá entregar: um link de acesso ao dashboard funcional; acesso liberado para visualização pelos avaliadores; permissões abertas, sem necessidade de solicitação de acesso.
O link deverá permanecer ativo e funcional até o encerramento oficial do Desafio dos Dados. 

5. Explicação da lógica utilizada
A equipe deverá apresentar brevemente como foi o processo de construção, respondendo às questões a seguir.

A avaliação considerará não apenas a estética da solução, mas principalmente:
organização lógica e clareza analítica - informações organizadas estrategicamente, com visualizações adequadas para análise rápida;
usabilidade - recursos relevantes de interação e exploração dos dados;
interpretação dos dados e qualidade da tomada de decisão possibilitada pelo dashboard - identificação correta de atrasos, cálculos e priorização das entregas críticas.
